Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios? In my application I would use the live data from Twitter to connect other users. When a user login to Twitter, the application will gain access to all of the followers and the users being followed. Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically? Because JSON-P allows access to data that is outside of the application, which is what APIs basically do, it works well with JavaScript. This means it is easy to convert into JavaScript and call the data to be displayed however it is meant to be displayed.

Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc. I learned that if you try to work through a project without any breaks in between, then the project will not come out to be what you expected. Not taking breaks will cause a lot of stress that is very unnecessary. I also learned to plan ahead when possible. Having a look at the future assignments can help you prepare for what's coming next. Creating a "to-do" list is also helpful and helps you keep track of what is due. Github is also very helpful when doing a project because you can track your progress and it allows others to view your work, so that they can give you pointers if needed. The main thing that I learned in design was that color makes a huge difference. If the colors are too dark, or too bright then the user may not enjoy using the application. Always think about how you felt when you saw an application that had color that was too bright, or too dark and how it made you feel.
